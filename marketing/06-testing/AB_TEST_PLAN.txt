A/B TEST PLAN
What to test, how to track, when to decide, what to implement

TEST 1: Paywall Threshold (3 vs 5 state views/month)
--------------------------------------------------------------------------------
Hypothesis: 3 views creates more urgency than 5
Variant A: 3 views/month
Variant B: 5 views/month
Metric: Paywall → trial start rate
Duration: 30 days
Sample size: 200+ users per variant
Decision rule: If difference > 20%, implement winner. If < 20%, keep testing.
Implementation: Backend config flag, easy to switch

---

TEST 2: Free Trial (No-CC vs CC-Required)
--------------------------------------------------------------------------------
Hypothesis: No-CC gets more trials; CC-required gets better conversions
Variant A: No credit card required
Variant B: Credit card required
Metric: Trial sign-ups AND trial → paid conversion
Duration: 60 days
Sample size: 100+ trials per variant
Decision rule: Compare total paid conversions (sign-ups × conversion rate)
Implementation: Stripe checkout flow modification

---

TEST 3: Pricing Presentation ($29/yr vs $2.42/mo)
--------------------------------------------------------------------------------
Hypothesis: Annual pricing feels like commitment; monthly feels accessible

Metric: Click-through on pricing CTA

RULES FOR TESTING
--------------------------------------------------------------------------------
1. Only run 1-2 tests at a time (don't muddy the data)
2. Run each test for at least 7 days (prefer 14)
3. Don't stop early unless one variant is clearly broken
4. Document everything in /ab-tests.txt
5. If a test wins, implement it immediately
6. If a test ties, pick the simpler/cheaper option
7. If a test loses, document why and move on
8. Never test more than one variable at a time
